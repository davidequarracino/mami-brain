# Contributing to mami-brain 🤖

Thank you! Please follow these technical steps to contribute:

1. **Setup**: Fork the repo, create a `.venv`, and `pip install -r requirements.txt`.
2. **Config**: Create a `.env` file with your `LLM_API_KEY`.
3. **Branching**: Use `feat/feature-name` or `fix/bug-name` for your branches.
4. **Standards**: All code comments, docstrings, and commits MUST be in **English**.
5. **Logic**: Define all data structures as Pydantic models in `api/schema.py`.
6. **PRs**: Ensure code is clean and modular before submitting a Pull Request.

**Current Priorities**: 
- **New Skills**: Adding more domestic functions/tasks to `api/schema.py`.
- **Reasoning**: Enhancing `core/planner.py` to handle complex multi-step requests.
- **Vision & Errors**: Image analysis and robust API error handling.

*Keep the code clean—Mami is watching!* 🧹
